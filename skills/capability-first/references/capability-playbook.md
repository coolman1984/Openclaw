# Capability-First Playbook

## 1. Classify the request
Ask:
- Is this strategy?
- Is this execution?
- Is this delegation?
- Is this review?
- Is this recovery?
- Is this optimization?

## 2. Check capability first
Look for an existing skill, tool, or workflow that already handles the request.

## 3. Route before reasoning
Choose the best existing capability and use it.

## 4. Delegate when needed
Split the work into clear sub-tasks and assign the right specialist.

## 5. Verify
Review the output and make sure it actually solved the request.

## 6. Promote repeats
If the same thing happens more than once, create:
- a skill
- a checklist
- a template
- an automation

## 7. Avoid waste
Do not:
- rebuild known workflows
- over-reason mechanical tasks
- forget existing capabilities
- use heavyweight reasoning when a skill or tool already fits

## 8. Finish cleanly
State:
- what capability was used
- what was accomplished
- whether anything is still pending
